## P00 / P0 / P1 / P2 / P3 Findings

### P00: Crude Market Binding Error Must Block Live

The Pulse report and review text both argue the `$200` strike No trade: No market price 93.95%, AI No 99%, edge +5.05pp. The executable plan and fill are instead bound to the `$115` strike No, filled at about 45.7%. These are different risk profiles: `$200` is an extreme tail, `$115` is a volatile but much less extreme threshold. The `$200` probability, edge, Kelly, and thesis cannot justify the `$115` fill.

Add a pre-execution validator: report title, rule threshold, marketSlug, tokenId, and outcomeLabel must match exactly; bestBid/bestAsk and decision price may differ within 3%. Different strikes under the same event must not bind by eventSlug alone.

### P1: 4/20 Deep-Research Coverage Is Not Live-Ready

The candidate pool had 20 markets and only 4 received rules + comments + orderbook research. This is acceptable for pre-scan, not for live trading input. Minimum live standard: every executable candidate must have complete deep research; if Top 3 is selected from 20, at least Top 10 or 60% of candidates should receive comparable research, otherwise the output should be a watchlist.

### P3: Sports Data Collection TODO (France Example)

The report itself says France lacks latest FIFA/Elo rankings, injuries, bookmaker odds, and sibling-market total-probability checks. This is a sports-data collection improvement, not a core blocker for this run. It has been added to the P3 TODO list in [`../backlog.en.md`](../backlog.en.md). Future sports-market runs should add these sources and sensitivity checks.

### P0: Existing Position Review Must Become Real Refresh, Not Stale-Hold Wording

All 6 existing positions had no fresh signal, while the review says “still has edge: 6/6” and individual rows also say `edge=0.0000`, `Pulse coverage none`, and `human review yes`. Correct wording should be `stale-hold / no fresh edge / needs human review`, and these should not count as active edge. The main follow-up is not same-direction Buy-No constraints; it is real review of each existing position: fresh evidence, adverse signals, stop/reduce conditions, and PnL attribution.

### P1: Run Summary Must Add PnL Analysis

The run summary records equity moving from $548.13 to $529.01 but does not explain whether the change comes from mark price, spread, fees, unrefreshed positions, fill valuation, or anomaly. Future summaries must include PnL analysis: immediate slippage/spread on new fills, mark-to-market changes on existing positions, fees/costs, and position-count anomalies. It also has 3 successful fills but position count increases by only 2; this should be an explicit audit warning.

### P2: Citations And Comment Evidence Can Be Stronger

The report has comment samples and categories, which is a good base. “Complete enough” should cover latest / top-liked / holder / opposing comments, short original quotes, holder status, and whether each comment changes probability. External citations should move from “source name” to “specific claim -> source -> credibility -> quote”.
